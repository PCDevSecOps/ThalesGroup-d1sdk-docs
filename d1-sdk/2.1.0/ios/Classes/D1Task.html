D1Task

public class D1Task

Main class for D1 SDK use cases.

Note

It is recommended to initialize D1Task as soon as the application is launched or the earliest possible time.

Note

Risk API contains session ID internally. The session ID is used by backend to identify the user’s activity. The lifecycle of session ID is linked with D1Task lifecycle, i.e. one D1Task instance use same session ID for all the use cases.

    Declaration

    Swift

    public func addDigitalCardToScheme(_ cardID: String,
                                   tokenRequestor: TokenRequestor,
                                   appURL: String,
                                   termsAndConditionsAccepted: Bool,
                                   completion: @escaping (String?, D1Error?) -> Void)

    Parameters

    cardID

    card identifier
    tokenRequestor

    the target token requestor that user has selected.
    appURL

    Your app’s custom URL scheme that the Token Requestor calls back upon completion of tokenization flow to return the user to where they started their journey.
    termsAndConditionsAccepted

    Terms and conditions accepted by user. Only applicable for Visa card scheme. Value set by the issuer to indicate to the token requestor whether or not the customer already accepted the issuer terms and conditions up-front. Required for Secure Remote Commerce. If true, the token requestor can suppress displaying terms and conditions to the customer and just submit the terms and conditions GUID to Visa as though the customer has already accepted the terms and conditions.
    completion

    The callback invoked when the operation is completed. When it succeeds, it returns push provisioning URL String. Otherwise, it returns error details. The push provisioning URL String is used to open and pass the information the Token Requestor App.
