RiskParams

public class RiskParams

An object that encapsulates all parameters required for risk assessment.

Category - Risk API

Note

Note:

  • For UIKit application, integrator is recommended to use init(view:placementName:placementPage:) to initialize the parameter.
  • For SwiftUI application, integrator is recommended to use init() or init(placementName:placementPage:) to initialize the parameter. Then, registerSingleView(_:) is used to register individual UI component.

    Parameters

    view

    The current view getting the events. On most cases, this is UIViewController. view.
    placementName

    The placement name to identify the flow associated with the current page. Supported values (so far) are:

    • “CreateAccountMobile”, for user registration page
    • “LoginMobile”, for login page
    • “TransferFundsMobile”, for transfer funds page
    • “GenericMobile”, for generic page
    placementPage

    The page number within the current flow. For single page flow, the value should be 1. For flow that spans multiple page, the value is the page number (1,2,3,…).

  • registerSingleView(_:)

    To register listener to UIView used in SwiftUI page.

    Note

    The listener is used for profiling user activity. SwiftUI components, e.g. TextField and Button, do not allow listener registration directly. For this reason, these components must be replaced with their UIKit equivalent, e.g. UITextField and UIButton. UIViewRepresentable pattern is used to allow access to the UIKit component.
