D1Task

public class D1Task

Main class for D1 SDK use cases.

Note

It is recommended to initialize D1Task as soon as the application is launched or the earliest possible time.

Note

Risk API contains session ID internally. The session ID is used by backend to identify the user’s activity. The lifecycle of session ID is linked with D1Task lifecycle, i.e. one D1Task instance use same session ID for all the use cases.

  • startAnalyze(_:completion:)

    Start monitoring of events for the current page. This method should be called at the start of the page or when the app resumes into foreground. e.g. viewWillAppear(_:), didBecomeActiveNotification:, SwiftUI onAppear(perform:).

    Category - Risk API

    Note

    Other Risk API pauseAnalyze(), stopAnalyze(completion:) should only be called after completion succeeds.

    Note

    Possible error: D1Error.Code.risk, caused by no internet connection, incorrect risk credentials.

    Parameters

    params

    parameter containing the details of the current page.
    completion

    block to be called when it finishes. It returns on the main thread. When it succeeds, it returns nil error. Otherwise, it returns error details.
  • pauseAnalyze()

    Pause the monitoring. This method should be called when the page navigates to the next page (during a multi-page scenario) or when the app goes into background. e.g. viewWillDisappear(_:), willResignActiveNotification.

    This API is only used in UIKit application. For SwiftUI, the pause is handled by RiskParams.unregisterSingleView(_:)

  • stopAnalyze(completion:)

    Stop monitoring the events for the current page. This method should be called to get the risk payload.

    Category - Risk API

    Note

    Integrator should send the risk payload to their backend, to be used in risk assessment call to D1 backend.

    Note

    Possible error: D1Error.Code.risk, caused by incorrect risk credentials.

    Declaration

    Swift

    public func stopAnalyze(completion: @escaping (inout Data?, D1Error?) -> Void)

    Parameters

    completion

    block to be called when it finishes. It returns on the main thread. When it succeeds, it returns risk payload. Otherwise, it returns error details. This completion returns risk payload in Data. The data is already in JSON format. Integrator can use JSON conversion method like jsonObject(with:options:) to convert to NSDictionary.
